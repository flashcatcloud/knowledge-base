在监控告警接入 Flashduty 时，有时原始告警信息可能不够丰富，或者格式不符合后续处理的要求。**标签增强**（Label Enrichment）允许您在告警数据入库前，对其进行二次加工。

通过标签增强，您可以从杂乱的描述中提取关键信息（如 IP、错误码），或者通过映射表补充业务元数据（如 `host-01` -> `owner:张三`），从而为后续的 **告警聚合**、**分派路由** 和 **故障分析** 打下坚实基础。

## 核心用途
---
标签是 Flashduty 处理告警的“原子单位”。丰富的标签能带来以下价值：

1.  **精准路由**：将不同 `service` 或 `region` 的告警分派给不同团队。
2.  **智能降噪**：作为告警聚合的维度（如按 `cluster` 聚合），减少噪音。
3.  **快速定位**：在故障列表中，直接展示 `instance_id` 或 `error_code`，一眼看清问题。


## 增强方式
---
Flashduty 提供四种增强模式，支持对 **告警标题**、**描述** 及 **现有标签** 进行处理。

### 1. 提取
使用正则表达式（RegEx）从长文本中“挖”出关键信息。
*   **场景**：监控系统只发送了一段文字描述 "CPU high on 192.168.1.10"，您希望提取出 `ip: 192.168.1.10` 作为独立标签。

### 2. 组合
将多个现有标签或固定字符串拼接成一个新的标签。
*   **场景**：您有 `region=cn-bj` 和 `host=web01` 两个标签，希望组合生成 `node_id: cn-bj.web01` 以便对接下游系统。
*   **语法**：支持 Go Template 语法，如 `{{.Labels.region}}.{{.Labels.host}}`。

### 3. 映射
相当于一个“翻译字典”。根据源标签的值，自动查表并生成新的标签。
*   **场景**：根据 `host` 自动补充 `owner`（负责人）或 `cmdb_id`。
*   **优势**：支持上传 CSV 文件批量管理映射关系，非常适合处理 CMDB 类数据。

### 4. 删除
移除不需要的标签。
*   **场景**：清理监控系统上报的无用元数据（如 `__internal_id__`），保持告警详情页整洁。

## 配置指南
---
进入 **集成中心 => 标签增强**，选择对应的集成即可开始配置。

### 通用选项
*   **限制条件**：您可以设置仅当满足特定条件（如 `level=Critical`）时，才执行增强规则。
*   **覆盖策略**：如果生成的新标签名已存在：
    *   **开启覆盖**：新值替换旧值。
    *   **关闭覆盖**：保留旧值，忽略新值。

![2025-09-22-13-51-32](https://docs-cdn.flashcat.cloud/images/png/35d7caf8b95fca29c660d1cc6fec6a40.png)

::: tip

标签规则可以配置多个，多个规则时会从上而下的顺序执行，当其中有规则不匹配时则不会生成/删除对应的标签，且没有提示信息。

:::

### 3. **标签映射** 
**场景：** 当源告警信息中的标签值不固定且不能直观定位其含义时可以通过映射的方式，将源标签映射新定义标签和值；比如源告警中只有资源类型ID信息，但希望将每个ID对应的资源类型名称也体现出来，这时可以通过映射实现。

#### 3.1 准备映射表文件（需要CSV格式）
以下为映射表文件示例，目的是将告警中的资源类型ID与实际的资源类型名称进行映射。

| ID | Type |
| :---: | :---: |
| A | server |
| B | router |
| C | gateway |
| D | database |
| E | MQ |

#### 3.2 创建映射表
1. 进入 `集成中心` -> `标签映射` -> `创建标签映射`。
2. 填写基本信息，如名称、描述、管理团队等。
3. 在 `映射表数据` 处，上传准备好的csv文件（如果数量较少，创建完成后，可以在映射详情页面中编辑添加）。
4. 选择`源标签`，如 `ID`，选择`目标标签`，如 `Type`。
5. 点击`创建`，完成映射表创建。

**附：映射表数据管理**

在映射表详情页面中，可以对映射表数据进行管理，包括新增、编辑、删除等操作。
1. 数据搜索：可以按照源标签的值进行搜索。
2. 数据添加：手动添加映射数据。
3. 数据上传：上传新的数据映射表，会覆盖已有数据。
4. 数据下载：下载当前映射表数据到本地。
5. 数据展示：展示当前映射表数据，可以编辑或删除。
