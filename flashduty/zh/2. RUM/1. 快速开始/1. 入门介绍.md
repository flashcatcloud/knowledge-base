## 什么是真实用户监控？

---

真实用户监控（RUM）是一项创新技术，它能够追踪并分析真实用户在使用您 Web 和客户端应用的实际体验。与传统的模拟测试不同，RUM 直接从用户侧采集数据，为您呈现应用在真实环境中的运行状况。

Flashduty RUM 让开发人员、运维工程师和业务相关方能够直观地了解应用性能，及时发现问题并持续优化用户体验，这一切都源于对终端用户数据的实时采集与分析。

### 核心优势

1. **快速定位问题根因**：通过会话查看器联动各类事件（页面访问、资源加载、用户交互等）进行多维度下钻分析，结合 trace 链路追踪打通前后端数据，快速定位从用户端到服务端的问题根源。
2. **全面提升应用稳定性**：自动捕获 JavaScript 报错、客户端 crash、网络故障等影响用户的问题，并提供丰富的上下文信息，在问题扩散前及时发现并解决。
3. **持续优化用户体验**：实时掌握页面加载时间、资源加载效率和 JavaScript 运行状况等关键性能指标，基于真实用户数据科学指导性能优化，告别主观臆测。
4. **数据驱动决策优化**：通过分析看板深入洞察 APP 崩溃情况、资源性能表现和数据变化趋势，为产品迭代和业务决策提供可靠依据。
5. **构建完整可观测体系**：与 Flashduty 监控体系深度集成，通过协作空间实现异常告警，打通前后端监控链路，建立统一的可观测性平台。

## 为什么选择 Flashduty RUM？

---

1. **开箱即用**：5 分钟快速接入，无需复杂配置，支持 Web、iOS、Android 多端统一监控，降低接入和维护成本。

2. **生态融合**：与 Flashduty 与 Flashcat 监控体系深度集成，通过协作空间实现异常告警，结合 trace 链路追踪，实现前后端全链路问题定位与分析，一站式解决可观测性需求。

3. **轻量高效**：SDK 采用轻量化设计，在保证数据采集完整性的同时将对用户体验的影响降至最低。

4. **灵活部署**：支持 SaaS 云服务和私有化部署两种模式，满足不同企业的数据安全和合规性要求。

## RUM 工作原理

---

Flashduty RUM 通过在您的 Web 应用或者 native(安卓、iOS)应用中植入轻量级 SDK 来实现数据采集。SDK 主要收集以下几类数据：

1. **页面访问**：记录页面加载过程、导航耗时以及用户环境信息。

2. **资源加载**：详细记录图片、脚本和接口调用等资源的加载情况。

3. **用户交互**：采集点击、表单提交等用户操作及自定义事件。

4. **异常信息**：记录 JavaScript 异常、网络故障等问题。

5. **长任务追踪**：监控可能造成页面卡顿的长耗时任务。

这些数据会实时传输到 Flashduty 后台进行处理和分析，最终以直观的仪表盘和报表形式呈现，帮助您全面掌握应用性能和用户体验状况。
